## 2. 목적

- 사용자가 약속 날짜 범위를 월 단위 캘린더에서 직관적으로 선택할 수 있어야 한다.
- 한국 사용자를 기준으로 요일 순서, 주말 색상, 공휴일 표기를 제공해야 한다.
- 선택 불가능한 날짜와 바깥 달 날짜를 시각적으로 명확히 구분해야 한다.

## 3. 레이아웃 요구사항

### 3.1 기본 구조

- 캘린더는 월 제목과 좌우 이동 버튼을 가진 상단 헤더를 포함한다.
- 월 그리드는 7열 고정 구조를 사용한다.
- 각 날짜 셀은 날짜 숫자와 필요 시 공휴일 이름을 함께 표시할 수 있는 2단 구조를 사용한다.
- 날짜 범위 중간 선택 영역은 가로 막대 형태의 배경으로 연결한다.

### 3.2 요일 배치

- 한 주의 시작 요일은 `월요일`이어야 한다.
- 요일 헤더 순서는 `월 화 수 목 금 토 일`이어야 한다.
- 요일 헤더 텍스트는 주말 여부와 관계없이 동일한 중립 회색 계열을 사용한다.

### 3.3 바깥 달 날짜 표시

- 특정 월의 1일이 첫 칸이 아니면 앞칸에 이전 달 마지막 주 날짜를 표시해야 한다.
- 특정 월의 마지막 날 이후 남는 칸에는 다음 달 날짜를 표시해야 한다.
- 바깥 달 날짜는 숨기지 않는다.
- 바깥 달 날짜는 회색 텍스트와 일반보다 가벼운 굵기로 표시한다.

## 4. 날짜 상태별 시각 규칙

### 4.1 일반 날짜

- 평일 날짜 숫자는 기본 남색 텍스트를 사용한다.

### 4.2 주말

- 토요일 날짜 숫자는 파란색으로 표시한다.
- 일요일 날짜 숫자는 빨간색으로 표시한다.

### 4.3 선택 불가능한 날짜

- 이미 지나 선택 불가능한 날짜는 회색 텍스트로 표시한다.
- 선택 불가능한 날짜는 주말 또는 공휴일 색상보다 회색 표시가 우선한다.

### 4.4 오늘

- 오늘 날짜는 기본 날짜보다 더 강조된 파란 계열 텍스트와 굵기를 사용한다.

### 4.5 선택된 날짜 범위

- 시작일과 종료일은 파란색 원형 강조 상태를 사용한다.
- 선택 범위 중간 날짜는 연한 회색 연결 바를 사용한다.
- 시작일과 종료일의 숫자 색상은 흰색으로 반전한다.

## 5. 공휴일 표기 규칙

### 5.1 공휴일 판별

- 한국 공휴일 데이터를 기준으로 날짜별 공휴일 여부를 판별한다.
- 공휴일은 주말 여부와 무관하게 날짜 숫자를 빨간색으로 표시한다.
- 다만 선택 불가능한 날짜 또는 바깥 달 날짜인 경우 회색 표시가 우선한다.

### 5.2 공휴일 이름 표시

- 공휴일인 날짜는 날짜 숫자 아래에 작은 텍스트로 공휴일 이름을 표시한다.
- 예시: `어린이날`, `현충일`
- 공휴일 이름은 빨간색 작은 텍스트를 사용한다.
- 선택 불가능한 날짜 또는 바깥 달 날짜의 공휴일 이름은 회색으로 표시한다.

### 5.3 공휴일 이름 축약 규칙

- 공휴일 이름이 `5글자 이하`이면 원문 그대로 표시한다.
- 공휴일 이름이 `5글자 초과`이면 앞 `3글자` 뒤에 `...`을 붙여 표시한다.
- 예시:
  - `어린이날` → `어린이날`
  - `석가탄신일` → `석가탄신일`
  - `대체공휴일예시` → `대체공...`

### 5.4 중복 공휴일

- 한 날짜에 여러 공휴일이 겹치면 현재 구현은 첫 번째 공휴일 이름만 표시한다.
- 다중 공휴일 병합 표기 방식은 현재 범위 밖이다.

## 6. 상호작용 요구사항

- 이전 달 버튼과 다음 달 버튼으로 월 이동이 가능해야 한다.
- 날짜 선택 모드는 `range`를 사용한다.
- 바깥 달 날짜도 표시되지만, 선택 가능 여부는 전달된 `disabled` 규칙을 따른다.
- 날짜를 선택하면 선택 범위에 맞춰 화면이 갱신되어야 한다.

## 7. 접근성 및 표현 원칙

- 월 이동 버튼은 `이전 달`, `다음 달` 접근성 라벨을 가져야 한다.
- 색상만으로 상태를 완전히 구분하지 않도록 선택 범위, 비활성, 바깥 달 상태는 굵기와 배경 차이도 함께 제공한다.
- 텍스트는 한국 사용자 기준 표현을 사용한다.
