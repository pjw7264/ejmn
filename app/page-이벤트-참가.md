## 1. 페이지 개요

### 1.1 페이지 목적

- 주최자와 참여자가 자신의 이름, 선택 비밀번호, 가능 시간을 등록하거나 수정한다.
- 이 문서는 별도 URL 페이지가 아니라 `/events/{eventId}` 상세 화면 안에서 열리는 참여/수정 패널의 요구사항을 다룬다.
- 저장 성공 후에는 `PATCH /api/events/{eventId}` 응답을 사용해 같은 상세 화면을 즉시 갱신한다.

### 1.2 대상 사용자

| 사용자 유형 | 설명 |
| --- | --- |
| 1차 사용자 | 이벤트를 막 생성하고 자신의 가능 시간을 입력하려는 주최자 |
| 2차 사용자 | 공유 링크로 들어와 기존 이벤트에 참여하려는 사용자 |
| 3차 사용자 | 같은 이름으로 기존 일정을 수정하려는 사용자 |

### 1.3 완료 기준

- 사용자가 이름과 가능 시간을 정상적으로 등록할 수 있어야 한다.
- 사용자가 기존 참여자일 경우 올바른 이름/비밀번호로 자신의 가능 시간을 수정할 수 있어야 한다.
- 프론트엔드는 `PATCH /api/events/{eventId}` 성공 후 최신 `EventDetail`로 상세 화면을 갱신해야 한다.
- 입력 UI는 서버가 허용하는 RRULE 범위를 벗어나지 않도록 제어해야 한다.

## 2. 공통 제약 및 정책

| 항목 | 규칙 |
| --- | --- |
| 시간 기준 | 입력은 사용자 현지 시간으로 보이더라도, API 전송은 UTC RRULE 규칙을 따라야 한다. |
| 참여자 식별자 | 참여자 수정 식별자는 `name` 이다. |
| 인증 방식 | `Authorization: Basic <base64(name:password)>` 를 사용한다. |
| 비밀번호 | 선택 입력이지만, 기존 참여자가 비밀번호를 가진 경우 수정 시 필요하다. |
| 이름 변경 | 기존 참여자의 이름 변경은 지원하지 않는다. |
| RRULE 제약 | 입력 결과는 `PATCH /api/events/{eventId}`가 허용하는 단일 RRULE 형식으로 직렬화 가능해야 한다. |
| 데이터 기준 | 패널은 먼저 `GET /api/events/{eventId}`로 이벤트 기간을 읽은 뒤 입력 범위를 제한한다. |
| 오류 처리 기준 | 입력 형식 오류와 인증 오류는 서버 응답 기준으로 표기한다. |

## 3. 사용자 목표와 핵심 시나리오

| 시나리오 ID | 사용자 목표 | 시작 조건 | 완료 조건 | 우선순위 |
| --- | --- | --- | --- | --- |
| S-01 | 내 일정을 처음 등록하고 싶다 | 상세 페이지에서 참여 패널을 열었다 | 저장 성공 후 상세 화면이 갱신된다 | `Must` |
| S-02 | 기존 일정을 수정하고 싶다 | 같은 이름으로 이미 참여 중이다 | 올바른 인증 후 수정이 반영된다 | `Must` |
| S-03 | 저장 실패 이유를 알고 싶다 | 인증 또는 RRULE 검증이 실패했다 | 오류 메시지를 보고 수정 후 재시도한다 | `Should` |

## 5. 정보 구조

### 5.1 화면 구성

| 영역 | 목적 | 포함 요소 | 우선순위 |
| --- | --- | --- | --- |
| 헤더 | 현재 액션 인지 | 패널 제목, 닫기 버튼 | `High` |
| 본문 | 이름과 가능 시간 입력/수정 | 이름 입력, 비밀번호 입력, 가능 시간 등록 컴포넌트 | `High` |
| 보조 정보 | 정책 안내 | 이름 정책, 비밀번호 정책, 30분 단위/RRULE 안내 | `Medium` |
| 하단 액션 버튼 | 저장 및 종료 | `저장하기` 버튼 | `High` |

### 5.2 구현 상세 규칙

1. 패널을 열기 전에 `GET /api/events/{eventId}` 결과가 있어야 한다.
2. 가능 시간 입력 컴포넌트는 이벤트의 `start`, `end`, `slotMinutes` 범위 안에서만 동작해야 한다.
3. 제출 시 이름/비밀번호로 Basic 인증 헤더를 만들고, 선택 결과를 단일 RRULE로 직렬화한다.
4. `PATCH /api/events/{eventId}` 성공 시 응답 `EventDetail`로 상세 화면 전체를 갱신한다.
5. `INVALID_MEMBER_AUTH`, `PASSWORD_REGISTRATION_NOT_ALLOWED`, `UNSUPPORTED_RRULE`, `RRULE_OUT_OF_EVENT_RANGE`, `RRULE_NOT_SLOT_ALIGNED` 오류를 각각 구분해 보여준다.
